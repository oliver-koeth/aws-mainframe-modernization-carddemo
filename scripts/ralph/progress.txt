## Codebase Patterns
- Keep all modernization scaffold artifacts under `output/`; do not place placeholder backend or frontend files elsewhere in the repository during Phase 0.
- Run backend quality checks from `output/backend/`; the scaffold is packaged as `app` with editable-install tooling defined in `output/backend/pyproject.toml`.
- On fresh machines, bootstrap backend checks with `python3 -m venv .venv` and `.venv/bin/pip install -e '.[dev]'`, then run smoke checks, pytest, and mypy through `.venv/bin/python`.
- Run frontend quality checks from `output/frontend/`; keep Angular workspace metadata local to that directory and use `npm run typecheck` (`ngc -p tsconfig.app.json`) as the baseline scaffold validation before build-specific stories land.
- Frontend build verification should run as `npm run build` from `output/frontend/`; Angular CLI writes the distributable bundle to `dist/carddemo-frontend`.
- The frontend shell uses standalone Angular bootstrapping: keep `src/main.ts` focused on `bootstrapApplication(AppComponent, { providers: [provideRouter(routes)] })`, with routes defined in `src/app/routes.ts`.
- Browser verification for frontend scaffold stories can run against `npm start -- --host 127.0.0.1 --port 4200`; install Playwright's Chromium once with `npx playwright install chromium` if the local cache is empty.
- Keep scaffold browser tests self-contained in `output/frontend/playwright.config.ts` by using a `webServer` array that starts both `../backend/.venv/bin/python -m uvicorn app.main:app` and `npm start -- --host 127.0.0.1 --port 4200` before Playwright hits the proxied Angular route.
- When validating Angular proxy wiring in Playwright, fetch relative `'/api/...` endpoints inside `page.evaluate(...)` after loading a routed page so the test proves browser-context proxying and JSON parsing together.
- Angular's Vite-based dev server proxy config should use `pathRewrite` in JSON proxy files; defining a raw `rewrite` object crashes request forwarding because Vite expects `rewrite` to be a function at runtime.
- Frontend routes should consume backend data through `src/app/api-client.service.ts`; keep the shared `'/api'` base path in that service instead of hard-coding proxy-prefixed URLs in components.
- Shared backend JSON persistence lives in `app/storage.py`; preserve typed `Decimal`, `date`, and `datetime` values by encoding them as `{"__type__": ..., "value": ...}` envelopes and always persist with same-directory temp-file replacement.
- Shared backend write coordination also lives in `app/storage.py`; use `write_store`, `write_schedules`, or `write_json_file` so updates acquire the sibling `.lock` file and fail with `StorageLockTimeoutError` on deterministic timeout instead of adding feature-level locking.
- Backend tests should share `FastAPI` app, `TestClient`, and disposable `StoragePaths` fixtures through `output/backend/tests/conftest.py` instead of recreating setup inline in each test module.
