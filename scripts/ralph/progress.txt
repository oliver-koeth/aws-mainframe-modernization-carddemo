- For flat-file modernization stories, document both `app/data/ASCII.seed` bootstrap sources and `app/data/ASCII` runtime-managed files; `init_gnucobol_data.sh` and `reset_gnucobol_data.sh` create or reset operational files like `dateparm.txt`, `dailytran_pending.txt`, and `tranrept_requests.txt` even when those files do not have seed counterparts.
- Runtime-managed imports still belong in `output/backend/app/seed_import.py`: read copybook-backed seed families from `app/data/ASCII.seed`, but source `tranrept_requests.txt` from `app/data/ASCII` and normalize a missing runtime log to an empty `report_requests` collection.
- The authoritative `store.json` root contract lives in `output/backend/app/models.py` `default_store_document()` and is enforced by `output/backend/app/storage.py`; future stories should add fields inside declared collections or bump schema version intentionally instead of reshaping the root object ad hoc.
- Keep `output/backend/app/models.py` `STORE_COLLECTION_NAMES` and `STORE_OPERATION_COLLECTION_NAMES` aligned with `default_store_document()` whenever `store.json` collections change; storage validation and bootstrap contract tests consume those constants as the canonical collection list.
- Storage-backed domain services should read raw collections through `app.storage.read_store()` and validate persisted rows into canonical Pydantic models at the service boundary before applying business logic; treat malformed persisted rows as deterministic consistency errors instead of passing untyped dicts deeper into services.
- Copybook-backed parsers under `output/backend/app/domain` should reuse `output/backend/app/fixed_width.py` for width validation, field slicing, text normalization, and signed-zoned-decimal decoding; keep record-family modules focused on field mapping and domain-specific code validation.
- Seed/bootstrap commands should run file rows through `output/backend/app/importing.py` `parse_lines_strict()` so malformed data hard-fails consistently with `SeedImportError.detail` (`source_name`, `line_number`, `raw_line`, `reason`) instead of inventing per-file quarantine formats.
- Extend Phase 1 seed bootstrap through `output/backend/app/seed_import.py` `SEED_SOURCES` and `bootstrap_store()` so every new imported record family shares the same CLI entry point, default repo seed directory, and `write_store()` persistence path.
- When a bootstrap story imports related record families, enforce required cross-file joins inside `output/backend/app/seed_import.py` before calling `write_store()` so structurally valid rows cannot persist broken references.
- GNUCobol `LINE SEQUENTIAL` flat files can be physically shorter than their copybook width because trailing spaces are dropped on disk; parsers should right-pad to the logical copybook width before field slicing and then validate required fields explicitly.
- Preserve leading-zero numeric identifiers from copybooks, such as customer IDs and SSNs, as strings in canonical JSON models unless the runtime semantics require arithmetic.
- Display-form `PIC S9(...)V99` flat-file amounts use a trailing overpunch character for both sign and final digit; decode that suffix before normalizing account or transaction money fields to `Decimal`.
- Transaction reference joins are composite: `CVTRA04Y` categories key on `transaction_type_code` + `transaction_category_code`, `CVTRA03Y` supplies the type lookup, `CVTRA01Y` category balances attach that pair to `account_id`, and `CVTRA02Y` disclosure groups attach the same pair to `account_group_id`.
- `tranrept_requests.txt` is a runtime-managed pipe-delimited log written by `CORPT00C`, not a fixed-width copybook file; parse it as `timestamp|user_id|report_name|start_date|end_date` and only expect `Monthly`, `Yearly`, or `Custom` report names.
- Keep `output/docs/record-layouts.md` as the top-level Phase 1 record-layout index and use the narrower `output/docs/*-records.md` files for detailed field mappings; update both layers only when the underlying source semantics change.
- Keep shipped seed baseline counts centralized in `output/backend/tests/test_seed_import.py` and mirror the same values in `output/backend/README.md`; intentional fixture changes should update both so bootstrap drift is visible in docs and CI.
- `card_account_xref`-driven inquiry services should preserve persisted row order: account/card lookups stop at the first matching xref to mirror GNUCobol sequential scans, while customer lookups can return all related xrefs/accounts/cards in that same order.
- The `COTRN02C` modernization service should resolve account/card inputs through `LookupService.lookup_card()` so transaction-add validation inherits the same first-match xref behavior, mismatch errors, and inactive-record checks as the inquiry flows.
- `COTRN02C` next-transaction ID assignment is order-sensitive: scan persisted `transactions[]` in store order, update the remembered value only when `transaction_id` is numeric, and increment the last numeric row instead of taking a collection-wide maximum.
- Phase 1 transaction creation should require the `(account_id, transaction_type_code, transaction_category_code)` row to exist in `category_balances[]` before appending a new transaction, so created events already map to a persisted posting bucket for later posting/balance stories.
- Keep bill-payment semantics split by source program: `COBIL00C` online bill payment appends a fixed payment transaction and zeroes only `accounts[].current_balance`, while `CBTRN02C` posting appends the posted transaction, updates the first matching `category_balances[]` row, and adjusts the account cycle bucket by transaction sign.
